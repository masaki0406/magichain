# Lobby画面設計方針（身内プレイ前提）

## 批判的評価（現状の課題）
- **没入感の破壊**: 盤面上に常時「参加/セーブUI」が載ると世界観が崩れる。
- **操作干渉**: ゲーム操作と管理操作が同一レイヤーにあり誤操作が起きやすい。
- **情報の階層が崩れる**: 初期導線（参加・再開）とゲーム進行が混ざり、認知負荷が高い。

## 方針
- **ロビー画面に分離**して「参加/引き継ぎ/セーブ管理」を完結させる。
- **盤面画面はプレイ専用**にし、没入感と操作集中を確保する。
- Spark（無料）運用のため、**Firestore直書き + ルール**で整合性を担保。

## ロビー画面の役割
1. ルーム参加
2. キャラ引き継ぎ
3. セーブスロット管理
4. ゲーム画面への遷移

## 最低限のUXルール（身内前提）
- 引き継ぎはメンバー全員可（確認ダイアログのみ）。
- セーブ/ロードは全員可（後でホスト限定に強化可能）。

## 今回の実装
- `/` をロビー画面に変更
- `/game` に盤面を配置
- Join/Save パネルは **overlay** と **panel** の2モードで使い分け

## 次の改善候補
- ホスト専用のロード権限
- キャラコード or ホスト承認
- ロビーでの準備完了/チャット
